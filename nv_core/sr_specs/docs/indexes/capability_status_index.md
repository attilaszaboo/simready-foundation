## Legend

The following criteria are used to calculate the capability status:

| Criteria | Points | Description |
|----------|--------|-------------|
| **Core OpenUSD Schema** | 45 | The required schemas are defined in the Core OpenUSD specification. |
| **Custom OpenUSD Schema** | 25 | The required schemas are defined in a custom OpenUSD specification. |
| **Custom OpenUSD Attributes** | 15 | The required attributes are specified as custom properties. |
| **List of Requirements** | 10 | The capability specifies a list of requirements. |
| **DCC Export (via Chaser)** | 5 | Documentation for DCC export via 'chaser script' is provided. |
| **CAD Export (via Chaser)** | 5 | Documentation for CAD export via 'chaser script' is provided. |
| **CAD Export (native)** | 5 | Export from CAD is possible. |
| **DCC Export (native)** | 10 | Export from DCC is possible. |
| **KIT Authoring** | 10 | Authoring in Omniverse/Kit is possible. |
| **Validator Docs Link** | 5 | The validator documentation is linked from the capability documentation. |
| **Concept Mapping Document** | 5 | Concept mapping documents are linked from the capability documentation. |
| **USD Exchange SDK Support** | 5 | The capability is supported by the USD Exchange SDK. |

### Status Thresholds

- **Complete**: 90+ points
- **Development**: 33+ points
- **Draft**: Less than 33 points
